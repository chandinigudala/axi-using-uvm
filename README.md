# AXI Protocol Verification using UVM

## Overview

This project implements a complete Universal Verification Methodology (UVM) based verification environment for the AMBA AXI protocol. The verification environment is designed to validate AXI read and write transactions using reusable, scalable, and coverage-driven verification techniques.

The project demonstrates industry-standard verification methodologies including constrained random verification, transaction-level modeling, scoreboarding, functional coverage, and SystemVerilog Assertions (SVA).

## Verification Components

### Transaction
- AXI Transaction Class
- Randomized Read and Write Operations
- Burst Configuration
- Address and Data Generation

### Sequence
- Write Sequence
- Read Sequence
- Random Sequence
- Burst Transaction Sequence

### Sequencer
Controls transaction flow between sequences and driver.

### Driver
Converts transaction-level objects into DUT pin-level signals.

### Monitor
Captures DUT activity and forwards transactions to the scoreboard and coverage collector.

### Scoreboard
Compares DUT outputs with expected results generated by the reference model.

### Reference Model
Predicts expected AXI protocol behavior for functional comparison.

### Functional Coverage
Collects coverage on:
- Read Transactions
- Write Transactions
- Burst Types
- Burst Length
- Address Alignment
- Response Types

### Assertions (SVA)
Checks protocol correctness including:
- VALID/READY Handshake
- Address Stability
- Data Stability
- Response Timing
- Protocol Compliance
